# Petrichor — 디자인 레퍼런스 브리프 (DESIGN-REFERENCES)

> 생성: 디자인/레퍼런스 리서치 워크플로우(6각도, 65개 레퍼런스). 기획의도(2000년대 동아시아 구도심 새벽·쪽타일·습도 슬라이더·작업용 고정 탭)를 코드로 옮기기 위한 디자인 근거.

## 핵심 디자인 방향
1. **리텐션 = "틀어놓고 잊는" 경험**(바이럴 아님). WindowSwap/Drive&Listen/Lofi Girl/myNoise 공통: 즉시 재생·최소 UI·무한 지속·방해 없음. → 첫 진입 즉시 풀스크린 재생, 컨트롤은 습도 슬라이더 중심 최소화.
2. **습도 슬라이더 = 마스터 uniform**. 단일 입력이 grainAmount·bloomStrength·chromaOffset·blurSamples·rainGain·reverbWet를 곡선으로 동시 구동(오디오 플러그인 매크로/모핑 UX 선례).
3. **그레이딩 = 단일 패스 셰이더**(ogl). 풀스크린 비디오 텍스처에 LUT(시안-틸: shadow→teal/midtone→orange)+시간축 그레인+비네팅+약한 색수차/스캔라인+contrast 축소(눅눅함)+Bayer 디더링을 한 패스에. three.js 다중 패스는 구형 노트북서 fps 저하.
4. **영상 = CORS-clean·faststart·심리스 루프**. 직접촬영(교대역) 원본을 ffmpeg xfade로 30-60s 심리스화, crossorigin=anonymous+정확한 ACAO로 WebGL taint 회피, provenance를 assets.license/source_note에 기록.
5. **오디오 = 레이어 믹서+ConvolverNode 리버브**. myNoise/A Soft Murmur식 개별 게인 레이어(빗소리·지하철·타이어)+습도 연동 리버브 wet.
6. **쪽타일 = 절제된 아이덴티티**. 로딩/프레임/여백에만, 콘텐츠를 가리지 않게.

## 안티패턴(피할 것)
- 바이럴 훅·과한 모션/UI·튜토리얼·강제 인터랙션(수동 소비 방해).
- 유튜브 임베드(ToS+CORS taint로 셰이더 불가).
- 글리치 데모式 과장 VHS(앰비언트 무드 파괴 — subtle 유지).
- 제너릭 스톡 영상(미학 미스+재배포/CORS 함정) → 직접촬영 우선.

# 부록: 6각도 리서치 원자료 + 레퍼런스 65건

## 각도: single-control-ux

**레퍼런스**
- **myNoise — slider-driven multi-layer soundscape generator (+ Animate mode)** (audio-plugin-macro) https://mynoise.net/
    - 잘하는점: The single closest functional precedent for Petrichor's audio side. Each slider = one isolated sound layer (rain patter, leaves, distant city hum), and presets/Animate move MANY sliders at once from one gesture — exactly the 'one input morphs the whole mix' model. Critically, it ships TWO calibration profiles per slider, a 'Low' and a 'High' position, and the actual playback level is interpolated between them; Animate's 'Soft' mode lands anywhere on that Low→High curve (good for natural sound), 'Hard' mode snaps to extremes (musical). This Low/High envelope per layer is precisely the curve abstraction the humidity slider needs: humidity=t blends each audio layer along its own personal Low→High curve rather than a global linear fade. It also added a dynamics compressor/limiter that engages when several sliders stack at high levels (red bars = limiting) — directly relevant because Petrichor's humidity sweep raises rain gain + reverb wet together and can clip.
    - 차용: Model the humidity slider as t∈[0,1] driving N independent layers, each defined by a (lowValue, highValue) pair plus a response curve, not a single shared fade. Steal the 'Soft vs Hard' idea as two grain/intensity presets. Add a master limiter/ducking stage because correlated gain+wet increases sum non-linearly — log/visualize when it engages (observability tie-in for the backend analytics).
    - 피할점: myNoise exposes all ~10 sliders at once (mixing-desk metaphor). Petrichor's thesis is the OPPOSITE — hide the N sliders behind ONE. Don't surface the per-layer faders to the end user; keep them as server-side/config curves and expose only humidity.
- **Nightfox Audio — MacroMacro (one knob → many effect params)** (audio-plugin-macro) https://pluginmax.com/product/nightfox-audio-macromacro/
    - 잘하는점: Cleanest articulation of the exact data model Petrichor needs. One macro knob is assigned to any combination of effect parameters, and EACH target gets its own three properties: range (how far it travels), curve (response shape), and direction (same-direction or inverse). That is the humidity slider's mapping table verbatim — rain gain (up, exponential), cyan tint (up, ease-in), grain (up, linear), reverb wet (up, S-curve), and e.g. high-frequency air/brightness (DOWN, inverse) as humidity rises.
    - 차용: Define the humidity mapping as a list of {target, range:[min,max], curve, direction} records. This is serializable JSON the backend can own/version (mood presets become rows), letting the frontend stay a dumb renderer that reads param values for a given t. Inverse direction is the key trick for making humidity feel real: some things rise (rain, reverb, grain) while others fall (treble sparkle, image clarity) at the same time.
    - 피할점: MacroMacro lets the user re-author the mapping live. Petrichor should ship curated, hand-tuned mappings per mood ('교대역 새벽') — the curve authoring is a design-time/backend concern, not a user-facing control.
- **Ableton Live — Macros + Macro Variations + 'morph' workarounds** (audio-plugin-macro) https://help.ableton.com/hc/en-us/articles/360019103480-Macros-and-Variations-FAQ
    - 잘하는점: The canonical macro-mapping reference: up to 16 macro knobs each mapped to multiple device parameters with custom per-parameter min/max ranges. The classic trick — map one knob to two params with opposite ranges (0→100% and 100→0%) — is the textbook way to get a crossfade/morph from a single control. Macro Variations capture full snapshots of all macro positions as named 'moods' you can recall.
    - 차용: Use the opposite-range trick for Petrichor's cross-mood transitions (dry day ↔ wet dawn). Treat each saved 'mood' (교대역 dawn, etc.) as a Macro-Variation-style snapshot of all curve endpoints — a natural backend entity (Mood presets table, Flyway-migrated, versioned).
    - 피할점: The well-documented Ableton pain point is that variation switching is ABRUPT and can't be automated/morphed natively (whole community asks for it). Petrichor must do the thing Ableton can't: interpolate continuously along t, so mood-to-mood and within-mood changes are smooth, never stepped.
- **Teenage Engineering — OP-1 / OP-Z color-coded encoders (Hick's Law)** (single-knob-hardware) https://teenage.engineering/products/op-z
    - 잘하는점: Design-philosophy anchor for the whole single-control thesis. Four color-coded endless encoders re-map contextually to control synth params, filters, effects, levels, and ADSR shape — radically fewer controls than the parameter count, deliberately leveraging Hick's Law (fewer choices = faster decisions, faster flow-state entry). This is the aesthetic justification for hiding N sliders behind one: it's not a limitation, it's the point.
    - 차용: Frame the humidity slider in portfolio/README as a Hick's-Law/flow-state design decision, not a missing feature. One expressive control = lower cognitive load = the 'leave it pinned in a work tab' retention goal. The single knob IS the product's restraint.
    - 피할점: TE encoders are MODAL (one knob means different things per page). Petrichor's humidity should be the opposite: ONE stable, always-means-the-same-thing control. Don't overload it with modes — its legibility is what makes it ambient/calming.
- **Datavized 'Morph' + EditArt — single-parameter morphing / slider possibility space** (generative-art-morph) https://flowingdata.com/2018/09/26/morph-an-open-source-tool-for-data-driven-art-without-code/
    - 잘하는점: Generative-art precedent for one value sweeping a whole visual field. Morph generates a series of transitional Bézier 'in-between' frames between two end-member states (you set two endpoints, it interpolates the morph). EditArt exposes sliders that each walk a parameter through its 'possibility space.' Together they validate the t∈[0,1] interpolation-between-two-keyframes model for the WebGL grading side.
    - 차용: Author Petrichor's WebGL look as two end-member grades — 'dry' and 'fully humid' LUT/uniform sets (cyan tint amount, grain density, bloom, contrast) — and let humidity Bézier/ease-interpolate uniforms between them in the shader. Same two-endpoint morph model the audio layers use, so ONE t drives both audio and visuals coherently.
    - 피할점: Generative-art morphs often aim for surprise/novelty (77 Million Paintings never repeats). Petrichor wants the opposite: predictable, repeatable, returnable mood. Make t deterministic — same humidity always yields the same look/sound — so the 'work tab' feels like a place, not a slot machine.
- **Brian Eno — 77 Million Paintings (ambient generative mood, slow morph)** (weather-mood-slider) https://en.wikipedia.org/wiki/77_Million_Paintings
    - 잘하는점: Reference for the AMBIENT pacing of a single-control mood system. Hand-made elements are slowly cross-faded with a coupled shifting soundscape, in low light, at a rate so slow repetition would take 400+ years — engineered for meditative, glanceable, background presence. Validates 'slow, coupled audio+visual morph = something you leave running,' which is exactly Petrichor's retention metric.
    - 차용: Add a slow idle drift on top of the manual humidity control (a gentle, myNoise-Animate-style automatic wander around the user's set point) so a pinned tab keeps breathing without input. Couple audio and visual on the SAME clock so they feel like one weather system, not two widgets.
    - 피할점: Eno's piece has no real user control (pure generative). Petrichor's differentiator is the tactile single control — keep the user's hand on the humidity slider as the primary verb; let drift be a subtle secondary layer, never a takeover.
- **Single-fader-along-a-curve mapping (Logic/VI-Control) + easing transfer functions** (curve-mapping-technique) https://vi-control.net/community/threads/logic-pro-an-alternate-way-to-control-multiple-plugin-parameters-with-a-single-fader-along-a-curve.139685/
    - 잘하는점: The concrete engineering recipe behind everything above: one fader controls multiple parameters where EACH parameter follows its own custom response curve, giving a single automation lane that coordinates many targets. Pairs with the easing-functions reference (Febucci) — transfer functions on [0,1]→[0,1], where quadratic/power curves read as 'smooth/unnoticed' and S-curves give natural ramps. This is the math for making humidity FEEL physical rather than linear.
    - 차용: Implement humidity as t→ each-target via a named easing function (linear, easeInQuad, easeOutCubic, smoothstep/S-curve). Rain gain ~ easeInQuad (barely there until it's humid, then floods); cyan tint ~ smoothstep; grain ~ linear; reverb wet ~ easeOutCubic (wet swells early). Store the curve name per target so curves are data, not hardcoded — backend-ownable and A/B-testable via analytics.
    - 피할점: Don't ship raw linear mappings for everything — simultaneous linear gain+tint+grain+wet reads as cheap/mechanical and is the 'thin CRUD' failure mode for the FEEL. The perceived quality lives entirely in per-target curve choice; budget design time there.

**시사점(takeaways)**
- Model the humidity slider as a single normalized input t∈[0,1] driving N independent targets, where each target carries its OWN {range:[min,max], curve, direction}. This exact data model is proven by Nightfox MacroMacro (curve/range/direction per param) and myNoise (Low/High profile per slider). It serializes cleanly to JSON, so the mapping/preset becomes a backend-owned, Flyway-versioned entity rather than frontend magic numbers — turning a 'pretty slider' into a real backend feature (mood presets, versioning, A/B analytics on curves).
- Use INVERSE direction for realism: as humidity rises, rain gain / reverb wet / grain / cyan tint go UP while treble sparkle and image clarity go DOWN simultaneously — the opposite-range trick from Ableton macros and MacroMacro's 'direction' flag. A weather system that only adds is unconvincing; coupled rise-and-fall is what sells the single control as physical.
- Per-target easing curves are where ALL the perceived quality lives — linear-everything is the 'thin/cheap' failure mode. Apply named transfer functions ([0,1]→[0,1]): easeInQuad for rain (latent then floods), easeOutCubic for reverb wet (swells early), smoothstep for tint, linear for grain. Store the curve name as data so curves are tunable/measurable, not hardcoded.
- Couple audio and visuals on ONE shared t and one clock, authored as two end-member states ('dry' vs 'fully humid' for both the audio layer set AND the WebGL grade), then interpolate — validated by Datavized Morph (two-endpoint Bézier morph) and Eno's coupled slow audio+visual cross-fade. This makes humidity feel like a single weather system, not two unrelated widgets.
- Frame the single control as a deliberate Hick's-Law / flow-state decision (Teenage Engineering OP-1/OP-Z), not a missing feature — and reinforce the retention goal ('pinned work tab') by adding a slow, optional idle DRIFT around the user's set point (myNoise Animate 'Soft' mode + Eno's glanceable ambient pacing). Keep the drift deterministic and subtle so the same humidity always returns the same mood — a place, not a slot machine.
- Protect the mix: correlated gain+wet stacking sums non-linearly and clips, so add a master limiter/ducking stage (myNoise added exactly this with a 'red = limiting' indicator). Log when limiting engages — a natural hook into Petrichor's intended observability/analytics backend depth, and a concrete metric to expose via Micrometer/Actuator.

## 각도: ambient-services

**레퍼런스**
- **WindowSwap** (Virtual window / fixed-frame ambient video) https://www.window-swap.com/
    - 잘하는점: One button ('Open a new window somewhere in the world') drops you into a real, user-submitted 10-min HD window view with native ambient sound and a tiny location+time label in the corner. The product IS a single frame you leave open — zero feed, zero dashboard. All submissions are human-reviewed for a consistent 'quietly looking out a window' tone. Retention layer: free account = bookmark favorite views + upload your own window; paid 'All Access' = search specific cameras instead of random shuffle.
    - 차용: This is the closest spiritual sibling to Petrichor's 'leave the tab open' goal. Borrow: (1) the single-button cold open that commits to a mood instantly (Petrichor's first mood = 교대역 station) with no menu; (2) the subtle corner label (location + time captured) as the ONLY chrome — for Petrichor a '교대역 · 04:12 · 습도 78%' tag reinforces the docu-real, self-shot premise; (3) the curation-as-quality moat (human review keeps tone coherent) maps directly to Petrichor's self-shot footage + tracked-source audio. Backend angle: bookmarking views and 'upload your window' are exactly the media-pipeline + auth + ownership features Petrichor wants depth in — model 'saved moods' and (later) user-submitted clips as first-class entities, not thin CRUD.
    - 피할점: WindowSwap's random shuffle can feel arbitrary and its account wall for basic saving frustrates passive users; don't gate the core leave-it-on loop behind login. Keep anonymous play instant; make auth additive (save/sequence/upload), never required to start.
- **Lofi Girl (lofi hip hop radio - beats to relax/study to)** (24/7 animated livestream / parasocial study companion) https://en.wikipedia.org/wiki/Lofi_Girl
    - 잘하는점: A single looping animated character studying in her room, ~40k concurrent viewers daily, hundreds of millions of views. Retention is parasocial: research/press describe the character as a 'partner' and 'safe companion you don't need to expend energy interacting with' — she symbolizes calm productivity. The unchanging loop is the point; consistency over novelty. Spawned a whole brand universe and a 7-year 'graduation' lore moment that drove press.
    - 차용: Petrichor's retention bet ('작업 탭 고정', not viral) is exactly Lofi Girl's. Borrow the parasocial-place idea: 교대역 at dawn can become a recurring 'character' (a place you return to like a companion), so consistency and a stable identity beat constant new content. Backend depth: the value isn't the video, it's the always-on session — instrument session-length/return-rate (Micrometer/Actuator) as the core KPI, and treat 'the stream never breaks' as an observability/reliability SLO (when Lofi Girl's stream was wrongly removed it was front-page news — uptime IS the product). Lightweight 'lore'/seasonal change can be modeled as a scheduling/sequencing backend feature.
    - 피할점: Lofi Girl's magic is brand + community built over years; a solo portfolio app can't manufacture that overnight. Don't over-invest in lore/mascot; invest in the reliability and session instrumentation that make 'always on' trustworthy.
- **Drive & Listen** (First-person POV city footage + live local radio) https://driveandlisten.app/
    - 잘하는점: Loads a city, rolls pre-recorded 4K dashboard-POV street footage (synced from YouTube) layered with real-time local radio plus optional toggleable city ambience (honks, chatter). Self-described as 'ambient presence' — not a game, map, or travel guide. Minimal controls: pick city, pick station, mute/unmute outside sound, regulate driving speed. Born from one founder's homesickness for Istanbul.
    - 차용: Validates Petrichor's exact format: first-person POV footage + independently mixable audio layers + a small set of mood/intensity controls. Borrow the layered-toggle model directly — Petrichor's Web Audio layer mixing + 습도(humidity) slider is the same idea as Drive & Listen's separable radio/ambience/speed. Backend depth: separating a base video track from independently selectable/streamable audio layers is a real media-pipeline problem (ffmpeg segmenting, per-layer S3/R2 objects, manifest assembly) — exactly the non-thin-CRUD work Petrichor wants. The 'regulate speed' control suggests a parametric playback engine worth modeling server-side (sequencing/state).
    - 피할점: Drive & Listen leans on third-party radio and YouTube-sourced footage; Petrichor's differentiator is self-shot video + provenance-tracked paid AI audio. Lean into owning the asset pipeline and source attribution (a backend/metadata strength) rather than aggregating others' streams.
- **A Soft Murmur** (Minimal ambient sound mixer (sliders-as-the-product)) https://asoftmurmur.com/
    - 잘하는점: The mixer IS the experience: a row of ~10 sounds (rain, thunder, waves, wind, fire, birds, crickets, coffee shop, singing bowl, white noise) each with a slider, and explicitly 'no feed, no dashboard, no newsletter trap, no social layer, no productivity philosophy.' It leaves you alone. The 'Meander' button gently randomizes non-zero sounds' volumes so the blend undulates like real life instead of looping flatly.
    - 차용: Two high-value steals. (1) Slider-as-core-interaction validates Petrichor's 습도 slider being the primary control surface, not a buried setting — make humidity the hero knob that re-grades shader + re-mixes audio in real time. (2) 'Meander' = subtle algorithmic variation to defeat loop fatigue. For Petrichor, model a server-driven or seeded sequencing engine that gently varies audio-layer levels / shader-grade params over time so a 'leave-it-on' session never feels like a 30-second loop — this is genuine backend sequencing/state logic, not a thin endpoint. The radical no-chrome stance is the gold standard for 'leave-it-on'.
    - 피할점: A Soft Murmur is almost too bare — no sense of place, narrative, or return-hook beyond the mix itself. Petrichor's edge is specificity of place/era (2000s East-Asian old-town dawn); don't strip so much that you lose the evocative identity that makes someone come back to *this* mood.
- **myNoise** (Calibrated generative soundscapes / power-user audio engine) https://mynoise.net/
    - 잘하는점: 10 color-coded frequency sliders (20Hz–20kHz) per generator, with a real calibration step that shapes output to your hearing/equipment/room. Power feature: combine sliders across generators into custom composite generators. Donation-supported (free; optional $5/mo or one-time donation for lifetime access), ad-free, privacy-respecting. Built/maintained solo by audio engineer Stéphane Pigeon — a credible solo-portfolio proof point.
    - 차용: myNoise shows how depth-on-the-audio-side becomes the moat and the brand. For Petrichor: (1) treat audio layers as composable, parameterized, user-savable presets — a 'save/share my mood mix' feature is real backend (auth + persistence + shareable preset entities + analytics on which mixes retain). (2) Calibration → Petrichor could offer a one-time 'tune to your room/headphones' or humidity-baseline step that personalizes the grade, a memorable onboarding that justifies an account. (3) Solo-maintainer + donation model proves a single backend-focused dev can ship something people pay for; frame Petrichor's portfolio story the same way.
    - 피할점: myNoise's calibration and 10-slider depth can overwhelm casual users; it's a power tool. Petrichor should hide depth behind one obvious control (humidity) and reveal advanced layer mixing only on demand — progressive disclosure, not myNoise's wall of sliders up front.
- **Noisli** (Productized sound mixer + Pomodoro + color, freemium SaaS) https://www.noisli.com/
    - 잘하는점: 28 mixable sounds with per-sound volume saved as named 'Combos', plus a color/background generator and a configurable Pomodoro timer (work/break lengths, session counts, auto-start). Clean freemium: free with limits; premium unlocks more sounds, offline, and unlimited saved combos. Shows the path from 'toy' to retained-utility SaaS.
    - 차용: Noisli is the template for turning an ambient toy into a sticky tool with a real account system. Borrow: (1) named, saved, syncable presets ('Combos') — concrete auth + persistence + sync backend work, and the natural unit for analytics ('which mood/mix correlates with long sessions/returns?'). (2) A focus timer transforms passive ambience into an active work ritual = the literal '작업 탭 고정' use case; a Petrichor 'work session' timer that gently shifts the dawn scene/audio as the session progresses is both a retention hook and a sequencing/observability feature. (3) Freemium tiering (free core, premium = more layers/offline/unlimited saves) is a clean monetization to model in the backend even if never charged.
    - 피할점: Noisli's color generator and feature sprawl dilute mood specificity. Petrichor should resist becoming a generic productivity dashboard; keep the 2000s-dawn atmosphere intact and let the timer serve the mood, not overwrite it with productivity UI.
- **Rainy Mood** (Single-loop rain ambience (the canonical 'leave-it-on' page)) https://www.rainymood.com/
    - 잘하는점: The internet's oldest/most popular rain generator — millions use it to sleep/study/relax. Near-zero UI: it just plays. Recently layered a daily-changing featured music track over the rain (a tiny daily-novelty hook on top of a stable core), and extended to paid mobile apps with extra features.
    - 차용: Proof that a single, dead-simple, instantly-on loop can retain at massive scale for over a decade. For Petrichor: a 'daily featured track over a stable base' is a low-cost return-hook worth copying — a stable 교대역 base scene with a small daily-rotating element (a different passing train, a new audio top-layer, a slightly different sky grade) gives a reason to return without breaking the leave-it-on calm. Backend: a scheduled daily-rotation/curation job (Redis + a scheduler) feeding a 'today's mood' endpoint is exactly the kind of sequencing/scheduling/observability surface Petrichor wants — and analytics on whether the daily element lifts return-rate is a clean, measurable experiment.
    - 피할점: Rainy Mood does one sensory thing (rain) with little sense of place or visual depth. Petrichor's multi-sensory WebGL+audio premise is richer, so don't under-build the experience — but DO match Rainy Mood's instant, frictionless start.
- **Chillhop Music** (24/7 lofi livestream as a brand/label with merch + community) https://chillhop.com/
    - 잘하는점: From a laundry-room livestream (first-ever 24/7 lofi stream, 2017) to an audiovisual universe anchored by a recurring raccoon mascot and a seasonal 'Essentials' compilation series (30+ editions). Strong handcrafted-visual identity; monetizes via vinyl/merch where the four seasonal vinyls physically assemble into the year's narrative house. Aesthetic consistency + seasonal cadence = long-term retention and identity.
    - 차용: Chillhop shows retention via seasonal cadence + coherent aesthetic identity rather than algorithmic novelty. For Petrichor: model 'seasons/editions' of moods (e.g., a 교대역 monsoon edition, a winter-dawn edition) as a content-release cadence — backend-wise this is versioned content packs, scheduled releases, and ownership/provenance metadata (which ties straight into Petrichor's tracked-AI-audio-source requirement). The provenance/attribution system (every clip + audio layer carries source, license, capture metadata) is a genuinely meaty backend domain that also doubles as portfolio-worthy 'media rights/metadata pipeline' work.
    - 피할점: Chillhop's merch/label business is far outside a backend-learning portfolio's scope. Take the seasonal-cadence and provenance ideas, not the e-commerce; don't let monetization theater distract from backend depth.
- **Focusmate / virtual coworking & body-doubling** (Live presence / body-doubling for accountability) https://www.focusmate.com/
    - 잘하는점: Productivity rises simply from another person being present ('body doubling'), especially for ADHD. Platforms range from scheduled 1:1 video sessions (Focusmate) to always-on 2D ambient-presence offices and cafe/library-imitating rooms with footage + background music. The retained loop is consistency and silent shared presence, not interaction.
    - 차용: This reframes Petrichor's retention thesis usefully: 'leave-it-on' can be reinforced by ambient *social presence* without chat. A very lightweight, privacy-safe signal — e.g., 'N other people are at 교대역 right now' as a quiet corner count — taps body-doubling's accountability pull while staying calm and non-viral. Backend depth: presence counts = real-time connection tracking (Redis pub/sub or Redis Streams, WebSocket/SSE sessions), concurrency metrics, and anonymized analytics — exactly the infra (Redis, observability, sessions) Petrichor's stack already targets, and a strong portfolio piece. Optional: anonymous 'who's here' as a P2 feature.

- **gskinner: Making Reverb with the Web Audio API** (Technical — convolution reverb wet/dry) https://blog.gskinner.com/archives/2019/02/reverb-web-audio-api.html
    - 잘하는점: Concrete ConvolverNode recipe: split signal into a DRY path (input -> output) and a WET path (input -> DelayNode pre-delay/early-reflection multitap -> ConvolverNode -> wet GainNode -> output), with a starting multitap wet gain of ~0.2. Generates the impulse response synthetically via an OfflineAudioContext rendering filtered, decaying noise (highpass+lowpass + attack/decay/release envelope) — no external IR file needed. Notes 'the less dry signal, the further into the reverberated space your sound is.'
    - 차용: This is your humidity slider's audio mapping. Map humidity -> wet/dry balance (and IR decay/lowpass): low humidity = mostly dry, close, crisp; high humidity = more wet, longer/darker tail = sounds smear into a damp reverberant 새벽 alley. Synthesizing the IR via OfflineAudioContext avoids shipping/encoding an IR asset (lighter media pipeline) and lets humidity re-render the IR's decay+filter live. Wet gain ~0.2 is a sane subtle-ambient default.
    - 피할점: Convolution is CPU-heavy on low-end devices; gate IR length by humidity and consider a cheaper algorithmic reverb fallback. Don't crossfade wet/dry linearly — use equal-power (see Smus/Stony Brook ref).
- **Web Audio API book (Boris Smus) — Volume & Crossfade ch.3/6** (Technical — equal-power crossfade & gain) https://webaudioapi.com/book/Web_Audio_API_Boris_Smus_html/ch03.html
    - 잘하는점: Canonical reference establishing that wet/dry and track-to-track crossfades should use an EQUAL-POWER curve, not linear: gainA = cos(x * 0.5*PI), gainB = cos((1-x) * 0.5*PI), which holds sin^2+cos^2=1 so summed perceived loudness stays constant (no dip in the middle). Also the standard createGain() volume pattern.
    - 차용: Use equal-power for BOTH (a) the humidity wet/dry knob and (b) seamless loop crossfades and (c) mood-to-mood transitions (교대역 dawn -> next mood). Implement gains with setValueAtTime + linearRampToValueAtTime on AudioContext.currentTime for sample-accurate, click-free fades. The 0.707 center / 1.414 peak relationship is the number to remember.
    - 피할점: Linear ramps on raw gain cause an audible mid-crossfade volume dip and clicks on loop seams — the most common amateur Web Audio mistake; equal-power fixes it.
- **veltman/loopify + Hivenfour/SeamlessLoop** (Technical — gapless looping libraries) https://github.com/veltman/loopify
    - 잘하는점: Small libraries demonstrating truly gapless loops via Web Audio AudioBufferSourceNode.loop (decode whole buffer, loop in the audio thread) — solving the gap/click that the HTML5 <audio> element can't, because <audio> lacks the timing precision. SeamlessLoop and Gapless-5 show crossfade-overlap variants (~25-50ms up to 0.5s) when a loop point isn't perfectly zero-crossing.
    - 차용: For Petrichor's continuous beds (rain, subway rumble), decode to AudioBuffer and use source.loop=true for true gaplessness; reserve crossfade-overlap only for layers whose loop points aren't clean. Drives an asset-prep rule for your ffmpeg media pipeline: trim/normalize loops to zero-crossings and a documented loop length, store loopStart/loopEnd as asset metadata so the client loops without guesswork.
    - 피할점: Never build the ambient beds on HTML5 <audio> loop — it gaps/clicks on every cycle. The choice of Web Audio AudioBuffer here is load-bearing for the whole 'eternal calm' effect.

## 각도: film-grain-shaders — 2000년대 동아시아 구도심 새벽 무드를 위한 WebGL/GLSL 필름 열화 그레이딩 패스 리서치. Petrichor 프론트는 ogl(three.js 아님)로 풀스크린 비디오 텍스처(직접촬영 교대역 영상)를 단일 포스트프로세싱 패스로 그레이딩한다. 핵심 미학 = 눅눅함/습기(humidity 슬라이더로 grain·bloom·blur 강도 동적 제어) + 2000년대 디지털캠코더/VHS 열화(시안-틸 컬러그레이딩, 색수차, 스캔라인, 디더링). 따라서 리서치 결론: (1) 단일 패스에 LUT 그레이딩+grain+vignette를 머지하라(Matt DesLauriers — three.js EffectComposer식 다중 패스는 구형 노트북에서 20fps로 떨어진다, ogl은 더 미니멀하므로 ping-pong FBO를 직접 관리). (2) grain은 시간축 애니메이션이 필수(mattdesl/glsl-film-grain의 3D noise frame 파라미터로 정적 노이즈가 아닌 '살아있는' 입자감). (3) 시안-틸 룩은 shadow→teal/midtone→orange 채널 분리이며 .CUBE LUT를 3D 텍스처로 굽거나 GLSL에서 직접 채널 매핑. (4) VHS/색수차/스캔라인/wobble은 humidity·시간대에 따라 약하게(subtle) — Petrichor는 글리치 데모가 아니라 앰비언트이므로 Alisavakis식 contrast 축소(col = lerp(0.5, col, contrast))로 다이내믹레인지를 눌러 눅눅함 표현. (5) 디더링/포스터화는 새벽 저조도 그라데이션 밴딩을 의도적 질감으로 전환(Bayer 4x4/8x8). humidity 슬라이더 = grainAmount·bloomStrength·chromaOffset·blurSamples를 동시 구동하는 마스터 uniform으로 설계 권장.

    - 잘하는점: 3D noise 기반의 자연스러운 필름 그레인 glsl-module. 시그니처 grain(texCoord, resolution, frame, q)에서 frame이 3D noise의 Z축 오프셋이라 프레임마다 입자가 '움직여' 정적 hash 노이즈보다 사실적. resolution을 스케일해 grain 크기 조절, luminance 기반으로 밝은 영역의 grain을 줄이는(overlay/soft-light/additive blend) 권장 패턴 포함.

- **Maxime Heckel — The Art of Dithering and Retro Shading for the Web** (dithering / posterization) https://blog.maximeheckel.com/posts/the-art-of-dithering-and-retro-shading-web/
    - 잘하는점: White noise / Bayer ordered(2x2,4x4,8x8) / Blue noise 디더링과 컬러 양자화를 풀스크린 포스트 패스로 구현하는 프로덕션급 가이드. 양자화 공식 floor(color*(n-1)+0.5)/(n-1)을 RGB 채널별 적용, Bayer 인덱스를 screen coord modulo로 뽑아 threshold 비교하는 구체 GLSL 단계 제시.
    - 차용: 새벽 저조도 영상의 그라데이션 밴딩을 '버그'가 아닌 의도된 2000년대 디지털 질감으로 전환. Bayer 4x4 디더 + 약한 posterize(n=16~32)를 그레이딩 패스 끝에 두고, humidity 낮을 때(건조)만 살짝 켜 디지털캠코더 압축 질감 연출. 블루노이즈는 더 유기적이라 grain과 궁합.
    - 피할점: 풀 디더링/저색수 양자화는 Obra Dinn급 강한 스타일라이즈 — Petrichor는 앰비언트라 과하면 무드 깨짐. 미세하게(threshold 약하게), 또는 밴딩이 보이는 영역에만 적용.
- **Harry Alisavakis — Write-up: VHS Image Effect** (VHS / chromatic aberration / scanlines) https://halisavakis.com/write-up-vhs-image-effect/
    - 잘하는점: VHS 룩을 레이어별로 분해한 명확한 기술 글. (1)색수차+컬러블리드(X축 소오프셋 다중샘플, blue-yellow 분리), (2)스캔라인 step(0.5, frac(uv.y*lineAmount + time*speed)), (3)sine wave wobble, (4)grain col*=max(0.7,random), (5)tape dropout UV wrap, (6)contrast 축소 col=lerp(0.5,col,contrast). 각 레이어 파라미터 명시.
    - 차용: contrast 축소(lerp to 0.5)가 Petrichor '눅눅함'의 핵심 — 습한 새벽의 낮은 다이내믹레인지를 정확히 재현. 색수차는 화면 가장자리에만 약하게(비네팅과 결합), scanline은 거의 안 보일 강도로. 모든 레이어 파라미터를 humidity·시간대 uniform에 매핑해 동적 무드 제어.
    - 피할점: 풀 VHS(wobble+dropout+강한 스캔라인)는 글리치 데모 미학 — 바이럴 아닌 리텐션이 목표인 앰비언트엔 과함. wobble/dropout은 끄거나 극히 드물게, 색수차·contrast축소·미세 grain만 상시.
- **Building a Real-Time Dithering Shader (Codrops)** (dithering (구현 튜토리얼)) https://tympanus.net/codrops/2025/06/04/building-a-real-time-dithering-shader/
    - 잘하는점: Bayer matrix ordered dithering을 깔끔하고 합성 가능한(composable) 드롭인 포스트 패스로 만드는 최신(2025) 단계별 튜토리얼. Post Processing 라이브러리 기준이지만 GLSL 코어 로직은 엔진 비종속.
    - 차용: Codrops 튜토리얼의 Bayer threshold 텍스처/함수를 ogl 그레이딩 패스의 디더 모듈로 이식. Maxime Heckel 글과 교차검증해 4x4/8x8 매트릭스 상수와 인덱싱을 그대로 차용.
    - 피할점: Post Processing(pmndrs) Effect 래퍼 코드는 ogl에 안 맞음 — GLSL 함수만 추출해 쓸 것.
- **hughsk/glsl-dither (GitHub)** (dithering (glsl-module)) https://github.com/hughsk/glsl-dither
    - 잘하는점: C64 셰이더 데모 유래의 Bayer 2x2/4x4/8x8 디더링 glsl-module. dither(uv, color) 형태의 작은 함수들로 즉시 import 가능, 검증된 임계 매트릭스 상수 제공.
    - 차용: 디더 매트릭스를 직접 구현하지 말고 이 모듈의 검증된 함수를 그레이딩 패스에 import. mattdesl/glsl-film-grain과 같은 glsl-module 생태계라 ogl/vite 번들에 함께 묶기 좋음.
    - 피할점: glslify 전제 모듈 — Vanilla Vite+TS 빌드면 glslify 플러그인 추가하거나 함수 본문을 인라인 복사해야 함.
- **VHS Scanlines & FX (ShaderToy)** (VHS / scanlines / grain (레퍼런스 데모)) https://www.shadertoy.com/view/dtGyzR
    - 잘하는점: 필름 그레인 + 수직 밴드 + 약한 글리치 + 스캔라인을 결합한, 의도적으로 'subtle'한 VHS 스캔라인 데모. 과하지 않은 강도 튜닝의 참고점.
    - 차용: Petrichor가 원하는 '거의 안 보이지만 무의식에 깔리는' 열화 강도의 기준 데모. iChannel 비디오 입력→그레이딩 흐름을 ogl 비디오 텍스처 경로로 1:1 매핑해 강도값 출발점으로 사용.
    - 피할점: ShaderToy 특유의 iResolution/iChannel/mainImage 규약은 ogl로 옮길 때 vUv/uResolution/uTexture로 수동 포팅 필요.
- **Improved film grain (ShaderToy)** (film-grain (레퍼런스 데모)) https://www.shadertoy.com/view/3sGSWV
    - 잘하는점: 개선된 필름 그레인 ShaderToy 데모. luminance 반응형 grain과 컬러 grain 변형 등 mattdesl 모듈을 보완하는 대안 구현 비교군.
    - 차용: mattdesl 3D-noise grain과 비용/룩을 A/B 비교해 모바일 성능 예산에 맞는 쪽 선택. 컬러 grain(채널별 다른 노이즈) 아이디어를 시안 틴트 grain으로 변형 가능.
    - 피할점: ShaderToy 데모는 라이선스/출처 표기 확인 필요(Petrichor는 에셋 출처 추적 원칙) — 코드 차용 시 주석으로 출처 기록.
- **VHS tape effect (ShaderToy, Ms3XWH)** (CRT/VHS (레퍼런스 데모)) https://www.shadertoy.com/view/Ms3XWH
    - 잘하는점: 스캔라인 생성, 노이즈, 색수차(엣지 컬러 분리)를 한데 묶은 인기 CRT/VHS 데모. 색수차+스캔라인+노이즈 결합 패턴의 좋은 참고.
    - 차용: 엣지 기반 색수차 강도 곡선(중앙 0 → 가장자리 최대)을 비네팅 마스크와 공유하는 트릭을 차용해 가장자리만 눅눅하게 번지는 렌즈 질감 구현.
    - 피할점: 전형적 CRT 곡률/강한 스캔라인은 TV 미학이라 '카메라로 찍은 새벽 거리' 무드와 충돌 — 곡률·강한 스캔라인은 빼고 색수차·노이즈만.
- **Cinematic LUT Preview — WebGL Color Grading (toolkitgen)** (LUT / teal-orange 그레이딩 (도구/참고)) https://toolkitgen.com/tool/lut_previewer_webgl
    - 잘하는점: .CUBE LUT 파일을 브라우저에서 JS로 파싱해 3D 텍스처로 만들고 WebGL 셰이더로 즉시 적용하는 워크플로 설명. teal-orange = shadow의 blue 채널 + midtone의 red 채널 조작이라는 원리 명시.
    - 차용: 디자이너가 DaVinci/Premiere용 .CUBE를 그대로 만들고, Petrichor가 이를 3D LUT 텍스처로 구워 그레이딩 패스에 물리는 파이프라인. '교대역' 등 무드별 .CUBE 교체로 룩 확장. 시안-틸 원리(shadow→teal)를 LUT 없이 GLSL 직접 매핑하는 폴백도 가능.
    - 피할점: 3D LUT 텍스처는 WebGL2(texture3D) 또는 2D 타일 LUT(텍셀 정렬 주의) 필요 — WebGL1 폴백이면 8x8 타일형 64³ LUT를 2D로 펼쳐 수동 보간.

**시사점(takeaways)**
- 아키텍처 결정: 다중 포스트 패스를 쌓지 말고 LUT그레이딩+saturation+vignette+grain+디더를 가능한 한 단일 fragment 패스로 머지하라(Matt DesLauriers — 다중 패스가 구형 하드웨어에서 20fps로 추락). 단 bloom/blur는 이웃 픽셀 의존이라 별도 다운샘플 패스 + ping-pong FBO 2장 필요. ogl은 추상화가 얇아 이 FBO 관리를 직접 해야 함.
- humidity 슬라이더 = 마스터 uniform 설계: uGrainAmount, uBloomStrength, uChromaOffset, uBlurSamples, uContrast(lerp-to-0.5 양), uDitherStrength를 하나의 0~1 습도값에서 파생. '눅눅할수록' grain 거칠게 + bloom 번지게 + contrast 눌러 다이내믹레인지 축소(Alisavakis의 col=lerp(0.5,col,contrast)가 핵심) + 가장자리 색수차 강화.
- film grain은 반드시 시간축 애니메이션(mattdesl/glsl-film-grain의 3D noise frame=uTime). 정적 hash 노이즈는 죽은 화면처럼 보임. luminance 기반으로 밝은 영역(가로등/네온) grain을 줄여 soft-light 블렌딩.
- 시안-틸 룩은 두 경로: (A) 디자이너가 .CUBE LUT 제작→3D 텍스처(WebGL2)/2D 타일 LUT(WebGL1 폴백)로 구워 적용(무드 교체=PNG/LUT 교체로 확장), (B) GLSL 직접 채널 매핑(shadow→teal, midtone→orange). 포트폴리오 관점에선 (A)가 '미디어 파이프라인'다운 데이터 주도 설계라 더 인상적.
- 톤 절제가 미학의 핵심: Petrichor는 글리치/바이럴 데모가 아니라 리텐션형 앰비언트. VHS wobble/dropout/강한 스캔라인/CRT 곡률은 끄거나 극미세하게. 상시 켜는 것은 색수차(가장자리만, 비네팅 마스크 공유), 미세 grain, contrast 축소, 약한 디더(밴딩 영역). Bayer 4x4 디더+가벼운 posterize는 새벽 저조도 밴딩을 의도된 2000년대 디지털 질감으로 전환.
- 구현 경로(ogl): 비디오 텍스처를 매 프레임 texImage2D로 갱신 → addPass로 그레이딩 패스 연결이 최단 경로. ShaderToy 데모(dtGyzR, 3sGSWV, Ms3XWH)는 iResolution/iChannel/mainImage 규약을 vUv/uResolution/uTexture로 수동 포팅. glsl-module들(mattdesl/glsl-film-grain, hughsk/glsl-dither)은 glslify 전제이므로 Vanilla Vite엔 vite-plugin-glslify 추가하거나 함수 본문 인라인. 차용 셰이더는 출처 주석 필수(Petrichor 에셋 출처 추적 원칙).

- **Greg Girard — HK:PM (Hong Kong Night Life 1974–1989) & Kowloon Walled City** (photographer) https://www.creativeboom.com/inspiration/greg-girards-nocturnal-photographs-of-hongs-kongs-nightlife-in-the-1970s-and-80s/

    - 차용: This is your WebGL grading north star: warm-amber highlights (sodium lamps) against cyan/green shadow tint, heavy color saturation in light sources but desaturated mid-tones. Build a LUT that preserves blown-out neon cores while pushing shadows toward teal-green. The 'one light source = neon sign' logic should drive how the shader vignettes/pools light around emissive points rather than lighting the whole frame evenly.
    - 피할점: Don't over-grade into cyberpunk magenta-overload; Girard's restraint (one dominant hue per frame) is the lesson — pick ONE neon accent per mood, not a rainbow.

Color replaces dialogue ('show not tell'). This is the canonical reference for using grading itself as the emotional carrier, which is precisely Petrichor's thesis (humidity slider + shader grading = mood, no words). Academic breakdown: academia.edu/20268327 (Color Design in the Cinema of Wong Kar-wai).
    - 차용: Two concrete mechanics for the humidity slider / mood system: (1) the 'smudge' look = motion-blur + chromatic bleed on highlights, achievable as a WebGL pass (slight RGB channel offset + directional blur on bright pixels). (2) Adopt the 'one cool base + one warm accent' rule per mood-state: cyan/green base for the wet-dawn default, a single warm red/amber emissive accent that the user can intensify. Map the humidity slider to bleed/bloom radius, not just saturation.
    - 피할점: WKW's signature reds belong to interiors/passion — don't let red dominate the exterior dawn moods or you lose the 'damp blue morning' identity. Reserve warm accents as sparing punctuation.

The deliberate motion-blur on rain at slow shutter is a cheap, high-impact WebGL effect for the 'humidity high' end of the slider (animated streak overlay + bloom).
    - 피할점: His palette skews high-saturation cyberpunk and very contemporary Tokyo — pull it toward 2000s grain and lower saturation so it reads as nostalgic Korean/HK old-town, not present-day neon-game promo art.

    - 차용: His muted-blue + foggy-grey + dim-yellow triad is a calmer, lower-energy alternative grade to Girard's neon — perfect for a 'low humidity / quiet' end of your mood slider.     - 잘하는점: Codifies the 'familiar-but-empty, recently-vacated' feeling: transitional spaces shot when naturally empty (early morning), cool fluorescent lighting, late-20th/early-21st-c architecture, slightly degraded image quality reading as 'a brain's recall of the past.'     - 피할점: City pop can tip into bright 80s sunshine/retro-funk cheer — pull toward the melancholy/solitude end (the crosswalk-alone, apartment-at-night frames), not the upbeat neon-party end, to stay true to the '눅눅한 새벽' brief.

    - 잘하는점: Catalogs the early-2000s East-Asian-inflected look: Japanese graphic-design influence, Lomo cross-processing (saturated high-contrast 'soft' photos), bleach-bypass, and blur techniques; the 20-year nostalgia cycle that makes 2000s imagery resonate now.     - 잘하는점: Defines the exact light condition Petrichor lives in: sun just below the horizon at dawn, even low-contrast scattering, deep blue/cyan/purple sky, and 'after rain = jackpot' wet-street reflections.
