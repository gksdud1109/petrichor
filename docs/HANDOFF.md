## 1. 프로젝트 한 줄 정의
2000년대 동아시아 구도심의 눅눅한 새벽을 1인칭으로 거니는 가상 산책/드라이브 분위기 웹앱. 이름은 **Petrichor**(잠정), 비주얼 코어는 **쪽타일**(구도심 외벽의 흰 세로 타일 그리드).

## 2. 절대 잊으면 안 되는 제약 (요약 — 상세는 문서)
- **작성자는 백엔드 주니어. 이 프로젝트는 백엔드 학습 습작이다.** 프론트엔드(WebGL/Web Audio)는 '충분히 괜찮은' 선에서 **타임박스**하고, 백엔드 깊이(미디어 파이프라인·인증·분석 파이프라인·시퀀싱·관측성)를 **의도적으로** 구현한다. "예쁜 사이트 + 얇은 CRUD 백엔드"는 이 프로젝트의 **실패 정의**다.
- **유튜브 임베드 금지 → 자체 호스팅 짧은 루프 영상.** (이유: 유튜브 ToS 위반 소지 + 크로스오리진 iframe은 WebGL 셰이더 그레이딩 불가. 자체 호스팅이어야 진짜 필름 질감이 나온다.)
- **음악 = AI 생성 연주곡**(저작권 안전, 실제 곡·스트리밍 API 사용 금지). **환경음 = Web Audio API 레이어 믹싱**.
- **습도 슬라이더가 시그니처 UX**: 단일 슬라이더가 [빗소리 게인 + 시안 틴트 + 그레인 강도 + 리버브 wet]를 동시 제어.
- **MVP = 무드 1개를 깊게.** 다도시 확장·계정·결제는 MVP 제외(계정·미디어 파이프라인·분석은 Phase 2에서 학습 목적으로 추가).
- **성공 지표 = 리텐션(작업 탭 고정), 바이럴 아님.**
- **추천 스택**: NestJS(TS) + PostgreSQL(Prisma) + Redis + BullMQ + ffmpeg + Cloudflare R2 + CDN. (자유 교체 가능 — 핵심은 백엔드 깊이 항목 구현)

## 5. 작업 원칙
- 프론트는 무한히 예뻐질 수 있는 함정이다. Phase별 프론트 시간을 정하고 초과하면 백엔드 깊이 항목으로 전환한다.
- 매 Phase의 산출물에 **백엔드 마일스톤**이 반드시 포함돼야 한다.
- 큰 작업은 먼저 계획을 제시하고 내 승인을 받은 뒤 실행한다.
