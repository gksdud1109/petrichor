# Petrichor — PRD (Product Requirements Document)

| 항목 | 내용 |
|---|---|
| **제품명(잠정)** | **Petrichor** — 비 갠 뒤 아스팔트 냄새. 비주얼 코어는 **쪽타일**(구도심 건물 외벽의 흰/미색 세로 타일) |
| **한 줄 정의** | 2000년대 동아시아 구도심의 눅눅한 새벽을 1인칭으로 거니는 가상 산책/드라이브 분위기 웹앱 |
| **문서 버전** | v0.1 (초안) |
| **작성일** | 2026-06-21 |
| **상태** | 기획 확정 전 / MVP 정의 단계 |
| **분류** | 감성 기반 앰비언트(작업용 백그라운드) 웹 서비스 |

> 본 문서는 과장 표현을 배제하고 의사결정의 **근거와 한계**를 함께 기록한다. "대박", "딸깍" 같은 낙관적 수사는 의도적으로 제거했다.

---

## 1. 문제와 기회

### 1.1 문제
특정 정서(2000년대 동아시아 구도심의 눅눅한 밤거리, 이른바 '세기말/새벽 감성')를 소비하고 싶은 수요가 있으나, 현실 제약(날씨·시간·체력·거주지)으로 직접 경험하기 어렵다. 기존 가상 산책/드라이브 서비스는 이 정서를 정밀하게 다루지 않는다.

### 1.2 기회 — 3개의 검증된 트렌드 교집합
이 제품은 **현재 동시에 상승 중인 세 트렌드의 교집합**에 위치한다. (근거는 부록 B 참조)

| 트렌드 | 시장 신호 |
|---|---|
| **가상 도피(escapism) + 작업용 BGM** | WindowSwap(순방문 500만+), Drive&Listen, Lofi Girl(연 ~350만 달러) 등으로 카테고리 수요는 구조적으로 검증됨 |
| **Y2K / 세기말 노스탤지어** | 2025–2026 한국 라이브 트렌드. 시대 불안·취업난이 정서적 동력 |
| **Liminal space / 도시적 멜랑콜리** | Gen Z(90년대 말~00년대 초 출생)가 '슬프고 텅 빈 향수'로 강하게 반응. 시티팝·도머·After Hours 미학과 연결 |

**갭:** '글로벌 대도시 일상'은 다수 존재하나, **2000년대 동아시아 구도심의 눅눅함**만 정밀 큐레이션한 서비스는 사실상 부재하다.

## 2. 타깃 사용자

| 세그먼트 | 설명 | 사용 맥락 |
|---|---|---|
| **1차: IT 직군(개발자·디자이너)** | '디지로그' 감성을 즐기는 작업자 | 작업 중 고정 백그라운드 탭 (노동요) |
| **2차: 세기말/인디 감성 소비층** | 쏜애플·검정치마·잔나비 등 한국 인디 모던록 청취자 | 새벽 감상, 휴식, 집중 |

핵심 사용 맥락은 **"틀어놓고 다른 일을 한다"** — 능동적 탐색이 아니라 수동적 분위기 소비.

---

## 3. 가치 제안

> 방구석에서 만나는 비 갠 뒤 새벽 2시의 구도심.
> 1인칭 도시 영상 + 필름 질감 그레이딩 + 음악/환경음 이중 레이어를, **'습도 슬라이더' 하나로 감정처럼 조절**한다.

차별점은 기술이 아니라 **큐레이션의 정밀함**과 **단일 감정 컨트롤(습도)** 경험이다.

---

## 4. 핵심 가설과 성공 지표

> ⚠️ **바이럴(인스타/틱톡 공유)은 KPI가 아니라 유입 채널이다.** vibe 제품의 바이럴은 1회성 스파이크 후 감쇠한다(예: Drive&Listen). 우리가 검증할 것은 **"작업용 고정 탭이 되는가"**이다.

| 가설 | 지표 | 1차 목표(예시) |
|---|---|---|
| **Activation** — 첫 방문에 빠져드는가 | 첫 세션 10분 이상 체류 비율 | ≥ 30% |
| **Retention(핵심)** — 다시 켜는가 | D7 재방문율(작업 탭 고정) | ≥ 15% |
| **Engagement** — 깊게 쓰는가 | 평균 세션 길이 / 습도 슬라이더 사용률 | 세션 ≥ 25분 |

이 지표들은 무드 1개의 **완성도**로 검증된다. 코스 개수로는 검증되지 않는다.

---

## 5. MVP 범위

원칙: **넓게(여러 도시) ❌ → 무드 1개를 깊게 ✅.**

### 5.1 In Scope (MVP)
- **무드 1개**: 예) "비 갠 뒤, 새벽 2시, 교대역 일대"
- 자체 호스팅 **루프 영상 3~5개**(심리스, 30~60초)
- **오디오 2중 레이어**: AI 생성 연주곡 2~3곡(루프) + 환경음(빗소리·지하철·타이어) 다중 레이어
- **필름 질감 그레이딩**(WebGL 셰이더): 그레인·시안 틴트·다운스케일 등
- **습도 슬라이더 1개**(시그니처): 한 컨트롤이 [빗소리 볼륨 + 시안 틴트 + 그레인 강도 + 리버브]를 동시 제어
- 웹 only, **로그인 없음**(제품 가설 검증 우선)

### 5.2 Out of Scope (MVP 제외)
- 다도시/다코스 확장, 사용자 계정·소셜 로그인, 결제/구독
- 유저 생성 콘텐츠(UGC) 업로드
- 모바일 네이티브 앱
- 추천/개인화 엔진

> 단, 위 일부(계정·미디어 파이프라인·분석)는 **백엔드 학습 목적상 Phase 2에서 의도적으로 추가**한다. → 기술 문서 참조.

## 6. 핵심 기능

### ① 비주얼 플레이어
- 자체 호스팅 루프 영상을 풀스크린 재생.
- **WebGL 셰이더로 실시간 필름 그레이딩**(그레인/시안/블룸/해상도 열화). CSS 필터만으로는 2000년대 질감이 안 나오므로 셰이더가 필수.
- 비주얼 아이덴티티: **쪽타일 그리드** 모티프(로딩/여백/UI 프레임).

### ② 오디오 레이어 믹서
- **음악 레이어**: AI 생성 연주곡(저작권 안전) 루프 재생.
- **환경음 레이어**: 빗소리·지하철·타이어 소리를 Web Audio API로 개별 게인 제어, 다중 믹싱.

### ③ 습도 슬라이더 (시그니처)
- 단일 슬라이더가 여러 파라미터를 곡선(curve)에 따라 동시 제어:
  - 빗소리/환경음 게인 ↑
  - 화면 시안 틴트·그레인 강도 ↑
  - 오디오 리버브(wet/dry) ↑
- "설정 10개"가 아니라 **"감정 1개를 돌린다"** 는 경험이 제품의 시그니처.

## 7. 주요 의사결정 로그 (근거 포함)

| # | 결정 | 근거 / 한계 |
|---|---|---|
| D1 | **유튜브 임베드 ❌ → 자체 호스팅 짧은 루프 ✅** | (a) 유튜브 ToS는 플레이어 가림·변형·오디오 대체를 금지 → 법적/약관 리스크. (b) 크로스오리진 iframe은 canvas/WebGL 픽셀 접근 불가 → **진짜 셰이더 그레이딩 불가**. (c) 짧은 루프는 정적 파일이라 CDN(R2/Bunny) 비용이 거의 0 → "스트리밍 서버 = 비용 폭발" 전제는 틀림 |
| D2 | **음악: 원곡/스트리밍 API ❌ → AI 생성 연주곡 ✅** | 쏜애플 등 실제 곡은 저작권. Spotify Web Playback SDK는 자유 리믹스 불가. AI 생성(Suno/Udio 상업 티어)이 유일하게 합법적 경로(IP/라이선스 조건 확인 필요) |
| D3 | **습도 슬라이더를 시그니처로 승격** | 단일 감정 컨트롤이 제품의 차별적 UX. 복제 난이도가 낮은 '꼼수'가 아니라 경험 설계에 투자 |
| D4 | **MVP는 무드 1개로 한정** | 가설(작업 탭 고정)은 완성도로 검증되며 코스 수와 무관. 범위 확장은 프론트 잡일만 늘림 |
| D5 | **성공 지표 = 리텐션, 바이럴은 채널** | vibe 제품의 바이럴은 1회성. 지속 가치는 충성 코어의 재방문 |
| D6 | **네이밍: Petrichor + 쪽타일 비주얼** | 'SplitTile'은 분할/그리드 UI로 오독되어 쪽타일 레퍼런스가 증발. 이름은 무드(Petrichor), 비주얼 코어는 쪽타일로 분리 |

## 8. 리스크와 완화

| 리스크 | 영향 | 완화 |
|---|---|---|
| **콘텐츠 확보 비용**(정밀한 구도심 1인칭 영상은 희소) | 높음 | 직접 촬영 / 소액 라이선스 / AI 영상 생성. MVP는 무드 1개라 소수만 필요 |
| **AI 오디오·영상의 상업 라이선스/IP 불확실성** | 중 | 상업 티어 약관 확인, 출처·라이선스를 DB에 필드로 기록(추적 가능성) |
| **트렌드 감쇠**(Y2K/liminal 열기가 식음) | 중 | 일회성 유행이 아닌 '작업 탭' 효용으로 고정 사용자 확보(브랜드화) |
| **수익화 천장이 낮음** | 중 | 기대치를 '브랜드/포트폴리오'로 설정. 후원/멤버십/굿즈/싱크로 한정 |
| **기술적 복제 용이성**(해자 없음) | 중 | 해자는 큐레이션 취향 + 원본 에셋 라이브러리 + 시그니처 엔진 |

## 9. 비즈니스 / 수익화 (현실 직시)

이 카테고리의 직접 수익화는 **약하다.** 참고 모델은 결제벽이 아니라 **Lofi Girl**(브랜드 + 후원/멤버십 + 굿즈 + 싱크 라이선스)이다. 인프라 비용이 거의 0이므로 소규모 충성 사용자만으로도 운영은 지속 가능하나, **벤처 규모 성장 기대는 비현실적**이다.

- 1차: 무료 + 후원(Buy Me a Coffee 류)
- 2차: 서포터 멤버십(추가 무드/커스텀 믹스)
- 잠재: 콘텐츠 싱크 라이선스, 굿즈

**포지셔닝 결론: '스타트업'이 아니라 '브랜드형 포트폴리오 프로젝트'로 본다.**

---

## 10. 차별화와 해자

- **차별화(O):** 하이퍼 니치 큐레이션(2000년대 동아시아 구도심) + 습도 슬라이더 UX + 쪽타일 비주얼 아이덴티티.
- **해자(제한적):** 기술 자체는 복제 가능. 방어선은 **취향·원본 에셋·시그니처 그레이딩/오디오 엔진**의 누적.

---

## 11. Non-Goals (명시적 비목표)

- 지도 기반 아카이빙/탐색 서비스가 아니다.
- 음악 스트리밍 서비스가 아니다.
- 대규모 UGC 플랫폼이 아니다.
- '집중력 향상(과학/웰니스)' 포지션(Endel/Brain.fm)을 직접 표방하지 않는다 — 우리는 감성/분위기다.

## 12. 오픈 퀘스천

1. 첫 무드를 무엇으로 할 것인가? (교대역 / 광명 / 홍콩 구도심 중 1)
2. 영상 소싱 방식 1순위는? (직접 촬영 / 라이선스 / AI 생성)
3. AI 오디오 생성 도구와 상업 라이선스 범위 확정.
4. 도메인/상표: Petrichor 가용성 확인 필요.

## 부록 A. 네이밍 노트
- **채택:** 이름 = **Petrichor**(무드·냄새), 비주얼 코어 = **쪽타일**(구도심 외벽 흰 타일 그리드).
- **폐기:** *SplitTile* — 'split+tile'로 오독되어 쪽타일 레퍼런스 전달 실패 + 'Tile' 상표/검색 충돌.
- 절충: 이름으로 무드를 깔고, 로고·배경·UI 질감으로 쪽타일을 전면화.
